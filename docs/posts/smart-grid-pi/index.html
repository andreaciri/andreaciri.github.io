Smart grid

2022-10-22 — Written by Andrea Cirigliano

Since few months, for the first time I live in a house with an electric heat pump. As I always do when I have some kind of new machinery I immediately leaf trough the manual (note: this does not apply to software, for which I read the documentation after several attempts).

Inside the manual there’s a section called smart grid. The idea is to tell the heat pump when there is free electricity, or when the electricity is cheaper, to raise the set point temperature. I imagined this as a thermal battery where the house stores energy when it’s convenient and releases it when necessary.

For once a smart feature seemed really smart, so I decided to make it work with my solar panel system. I have a SolarEdge inverter and after a quick search I found it supports Modbus communication over TCP. There are plenty of libraries on GitHub to read real-time data directly from the inverter. Unfortunately, my inverter is connected to the network via WiFi instead of ethernet, and SolarEdge recently disabled Modbus over WiFi for security reasons. So I had to use the SolarEdge APIs, making http requests to their cloud platform.
